MiniMax Algorhitm

Intro

MiniMax is an algorhitm used in game theory and decision making to find the optimal move, usually in a two-player setting zero-sum game. Connect-4 was the game of choice, but any game with perfect information is well-suited for the algorhitm - f.ex. Chess, Tic-Tac-Toe, Othello etc.

Thoughts and Optimizations

The algorhitm continously simulates moves and position, which mean an efficient way to represent and copy board states is necessary. For Connect-4, nested loops is fine, but what if it were Chess or Go? One way to improve on this would be to avoid copying alltogether and [make the move - evaluate - unmake the move]. This would be much better memory-wise with decent scalability.

But to achieve even faster operations Bitboards has become the standard for most grid games, where instead of using a matrix, the board is represented by bits. Bits can be operated on extremely quickly with the operators AND, OR, XOR, SHIFT which are single CPU instructions. Instead of (for row in) you can do (bb & (bb >> 1)) to check the whole board.

Another great aspect of this is win checks using shifts like so: m = bb & (bb >> direction) if m & (m >> (2 * direction)):return True. Here each direction corresponds to a shift value, no scan or loop is needed.
